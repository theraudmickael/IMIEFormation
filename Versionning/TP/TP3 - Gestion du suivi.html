Naviguez et ajoutez le suivi d'historique

Votre chef de projet vient de détecter un problème dans votre suivi de projet, en voulant tester une régression dans la version 1.0.
Cependant les sources décrivent une version 1.1, dont il ignore l'état de livraison.
Il vous demande de tagger les commits de l'historique afin d'identifier les versions 1.0 et 1.1...

Retournez au commit correspondant à la version 1.0.
Puis ajoutez un tag 1.0.0
Retournez au dernier commit du projet
Puis ajoutez un tag 1.1.0
