Modification de la structure

Le client souhaite ajouter une nouvelle fonctionnalité au projet. Afficher le copyright...

Ajoutez, donc avant l'heure de build, dans le fichier src/application.bat (en personnalisant avec les bonnes valeurs) : echo "Copyright company - 201x"
De plus modifiez la ligne echo "version 1" en echo "version 1.1"

Conservez un historique avec pour message "display copyright"

Ajoutez un fichier changelog.txt, avec un récapitulatif de modification par version (utilisez la command git log --oneline > changelog.txt )

Conservez un historique avec pour message "add changelog"

Enfin pour des besoins de qualité vous ajoutez un dossier "test" composé d'un fichier "mon_test.bat"

Conservez un historique avec pour message "add test script"


Le lead dev vous impose des modifications

Votre processus de développement intègre un service d'intégration continue, qui va automatiquement générer votre changelog.txt.
Il n'est donc pas nécessaire de le conservez dans l'historique.

Supprimez le fichier changelog.txt de l'outil de versionning

De plus, le fichier "test/mon_test.bat" est mal nommé, renommé le en "test/test_app.bat"

Conservez un historique avec pour message "fix test name"
