Création d'une branche d'expérimentation

Votre lead dev vous propose de basculer les sources sous une forme générique. Pour cela, vous devez modifier un grand nombre de lignes.

Afin de ne pas pertuber le développement principal, créez une branche nommée "dev".
Puis réalisez les modifications adéquates, dans le fichier src/application.bat :

### application.bat - 201x - votre nom ###
version = "2"
build_hour = "11H00"

echo "Build ${build_hour}"
echo "version ${version}"

Conservez un historique avec pour message "add var to script"

Bascule pour un correctif de dernière minute

Pendant que vous expérimentez cette solution, votre chef de projet détecte un problème dans le fichier test/test_app.bat

Basculez sur la branche principale (master).
Ajoutez le bon header dans le fichier test/test_app.bat semblable à celui du fichier src/application.bat
Pensez à incrémenter votre version en 1.2

Conservez un historique avec pour message "add partner name"

Suppression d'une branche

Quelques jours plus tard, un collègue vous évoque lors d'une pause-café, qu'il existe une librairie pour PowerShell qui faciliterait votre développement.

Afin de tester cette solution, créez une branche "powershell".
Puis créez un fichier "lib.sh" dans le dossier "src".

Conservez un historique avec pour message "add powershell lib"

Cependant le chef de projet vous indique que le client ne dispose pas de PowerShell sur les machines de production, et ne souhaite pas l'installer...

(Appelait le formateur pour validation)

Supprimez cette branche !

Intégration de la branche "dev"

Le lead dev valide votre branche "dev".

Mergez votre branche "dev" dans la branche principale (master).
